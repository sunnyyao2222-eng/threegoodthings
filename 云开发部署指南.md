# 微信小程序云开发部署指南

## 📋 概述

本项目已集成微信小程序云开发，用户数据将存储在云端数据库中，实现多端同步。

---

## 🚀 快速开始

### 1. 开通云开发

1. 打开微信开发者工具
2. 点击顶部菜单 **云开发** → **开通云开发**
3. 创建云开发环境（建议创建两个环境）：
   - `dev-xxx`：开发环境
   - `prod-xxx`：生产环境
4. 记录环境 ID（格式如：`cloud1-xxx`）

### 2. 配置环境 ID

修改 `src/api/cloud.ts` 文件：

```typescript
wx.cloud.init({
  env: 'your-env-id', // 替换为你的云开发环境 ID
  traceUser: true
})
```

### 3. 创建数据库集合

在微信开发者工具的**云开发控制台** → **数据库**中创建以下集合：

#### 集合 1：users（用户表）
```json
{
  "_id": "自动生成",
  "openid": "用户唯一标识",
  "unionid": "开放平台唯一标识",
  "nickname": "用户昵称",
  "avatar": "头像URL",
  "isVIP": false,
  "vipExpireAt": null,
  "streakDays": 0,
  "totalPoints": 0,
  "treeStage": 0,
  "createdAt": "创建时间",
  "updatedAt": "更新时间"
}
```

**索引设置**：
- `openid`：唯一索引

#### 集合 2：records（好事记录表）
```json
{
  "_id": "自动生成",
  "openid": "用户标识",
  "userId": "用户ID",
  "content": "好事内容",
  "category": "分类",
  "imageUrl": "图片URL",
  "voiceUrl": "语音URL",
  "aiFeedback": "AI反馈",
  "points": 5,
  "date": "2026-03-02",
  "isPublic": false,
  "createdAt": "创建时间"
}
```

**索引设置**：
- `openid` + `date`：组合索引
- `isPublic` + `createdAt`：组合索引（用于社区）

### 4. 上传云函数

在微信开发者工具中：

1. 右键点击 `cloudfunctions/user` → **上传并部署：云端安装依赖**
2. 右键点击 `cloudfunctions/record` → **上传并部署：云端安装依赖**

等待部署完成（首次部署需要几分钟）。

### 5. 配置云函数权限

在**云开发控制台** → **云函数**中，为每个云函数配置：

- **超时时间**：20秒
- **内存**：256MB
- **环境变量**：无需配置

### 6. 配置数据库权限

在**云开发控制台** → **数据库** → **权限设置**中：

#### users 集合权限
```json
{
  "read": "auth",
  "write": "auth"
}
```

#### records 集合权限
```json
{
  "read": "auth",
  "write": "auth"
}
```

## 🔄 切换到云开发模式

### 方式 1：完全替换（推荐）

将 `src/store/user-cloud.ts` 重命名为 `user.ts`：

```bash
# 备份原文件
mv src/store/user.ts src/store/user-http.ts

# 使用云开发版本
mv src/store/user-cloud.ts src/store/user.ts
```

### 方式 2：条件判断

在 `src/store/user.ts` 中添加环境判断：

```typescript
import { cloudUserApi } from '@/api/cloud'
import { userApi } from '@/api'

// 判断是否使用云开发
const USE_CLOUD = true

const api = USE_CLOUD ? cloudUserApi : userApi
```

## 🔐 安全规则

### 数据库安全规则

在**云开发控制台** → **数据库** → **安全规则**中配置：

```json
{
  "read": "auth && doc.openid == auth.openid",
  "write": "auth && doc.openid == auth.openid"
}
```

这确保：
- 用户只能读写自己的数据
- 必须登录才能访问

### 云函数安全

云函数自动获取用户的 `openid`，无需前端传递，防止伪造。

## 🐛 常见问题

### 1. 云函数调用失败

**错误**：`errCode: -1, errMsg: "cloud.callFunction:fail"`

**解决**：
- 检查云函数是否已上传
- 检查环境 ID 是否正确
- 查看云函数日志

### 2. 数据库权限错误

**错误**：`permission denied`

**解决**：
- 检查数据库权限设置
- 确保用户已登录
- 检查 openid 是否正确

### 3. 云函数超时

**错误**：`timeout`

**解决**：
- 增加云函数超时时间（最大 60 秒）
- 优化云函数代码
- 减少数据库查询次数

### 4. 本地调试云函数

在云函数目录下运行：

```bash
cd cloudfunctions/user
npm install
```

然后在微信开发者工具中启用**本地调试**。

## 🔄 数据迁移

### 从本地存储迁移到云端

创建迁移云函数 `cloudfunctions/migrate/index.js`：

```javascript
// 迁移本地数据到云端
exports.main = async (event, context) => {
  const { localRecords } = event
  const db = cloud.database()

  // 批量插入
  const tasks = localRecords.map(record => {
    return db.collection('records').add({
      data: record
    })
  })

  await Promise.all(tasks)

  return { success: true, count: tasks.length }
}
```
