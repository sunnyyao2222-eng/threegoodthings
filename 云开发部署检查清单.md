## 🚀 部署步骤

### 第一步：开通云开发（5分钟）

1. **打开微信开发者工具**
   - 打开你的"三件好事"小程序项目

2. **开通云开发**
   - 点击工具栏的"云开发"按钮
   - 如果是第一次使用，会提示开通云开发
   - 选择"开通"，创建一个新的云环境
   - 环境名称建议：`three-good-things-prod`
   - 等待环境创建完成（约2-3分钟）

3. **获取环境ID**
   - 云开发控制台打开后，在"设置" → "环境设置"中
   - 复制"环境ID"（格式类似：`cloud1-xxx`）
   - **重要**：记下这个ID，后面需要用到

---

### 第二步：创建数据库集合（3分钟）

在云开发控制台中，点击"数据库"标签：

1. **创建 users 集合**
   - 点击"添加集合"
   - 集合名称：`users`
   - 权限设置：仅创建者可读写

2. **创建 records 集合**
   - 集合名称：`records`
   - 权限设置：仅创建者可读写

3. **创建 achievements 集合**
   - 集合名称：`achievements`
   - 权限设置：仅创建者可读写

4. **创建 resonance 集合**
   - 集合名称：`resonance`
   - 权限设置：所有用户可读，仅创建者可写

---

### 第三步：上传云函数（5分钟）

1. **上传 user 云函数**
   - 在微信开发者工具左侧，找到 `cloudfunctions` 文件夹
   - 右键点击 `user` 文件夹
   - 选择"上传并部署：云端安装依赖"
   - 等待上传完成（会显示绿色的勾）

2. **上传 record 云函数**
   - 右键点击 `record` 文件夹
   - 选择"上传并部署：云端安装依赖"
   - 等待上传完成

3. **验证云函数**
   - 在云开发控制台，点击"云函数"标签
   - 应该能看到 `user` 和 `record` 两个函数
   - 状态显示为"部署成功"

---

### 第四步：配置数据库索引（2分钟）

在云开发控制台的"数据库"中：

1. **users 集合索引**
   - 进入 `users` 集合
   - 点击"索引管理"
   - 添加索引：
     - 字段：`openid`，类型：升序，唯一索引：是

2. **records 集合索引**
   - 进入 `records` 集合
   - 添加索引：
     - 字段：`userId`，类型：升序
     - 字段：`date`，类型：降序
     - 字段：`category`，类型：升序

3. **achievements 集合索引**
   - 进入 `achievements` 集合
   - 添加索引：
     - 字段：`userId`，类型：升序
     - 字段：`type`，类型：升序

---

### 第五步：更新前端代码（3分钟）

1. **配置环境ID**
   - 打开 `src/api/cloud.ts`
   - 找到第3行的环境ID配置
   - 将你在第一步获取的环境ID填入：
   ```typescript
   const ENV_ID = 'cloud1-xxx' // 替换为你的环境ID
   ```

2. **替换Store文件**
   - 将 `src/store/user-cloud.ts` 重命名为 `src/store/user.ts`（覆盖原文件）
   - 将 `src/store/record-cloud.ts` 重命名为 `src/store/record.ts`（覆盖原文件）
   - 将 `src/store/tree-cloud.ts` 重命名为 `src/store/tree.ts`（覆盖原文件）

3. **保存并重新编译**
   ```bash
   npm run dev:mp-weixin
   ```

---

### 第六步：测试功能（5分钟）

1. **测试登录功能**
   - 在微信开发者工具中，点击"预览"
   - 用手机微信扫码打开小程序
   - 首次打开会提示授权登录
   - 点击"允许"，查看是否成功登录

2. **测试记录提交**
   - 在首页输入三件好事
   - 点击"提交"
   - 检查是否成功保存

3. **验证数据存储**
   - 回到云开发控制台
   - 进入"数据库" → `users` 集合
   - 应该能看到你的用户记录
   - 进入 `records` 集合
   - 应该能看到刚才提交的记录

## 🔧 常见问题

### Q1: 云函数上传失败
**解决方案**：
- 检查网络连接
- 确保 `package.json` 和 `config.json` 文件格式正确
- 尝试右键选择"上传并部署：云端安装依赖（不校验域名）"

### Q2: 登录时提示"云函数调用失败"
**解决方案**：
- 检查 `src/api/cloud.ts` 中的环境ID是否正确
- 确认云函数已成功部署
- 在云开发控制台查看云函数日志，查看具体错误信息

### Q3: 数据无法保存
**解决方案**：
- 检查数据库集合权限设置
- 确认用户已成功登录（有 openid）
- 查看云函数日志中的错误信息

### Q4: 提示"request:fail url not in domain list"
**解决方案**：
- 在微信开发者工具中，点击"详情" → "本地设置"
- 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
