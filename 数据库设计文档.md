# 数据库设计文档

## 📊 数据库概览

使用微信小程序云开发的云数据库（基于 MongoDB）。

---

## 📋 集合设计

### 1. users（用户表）

**用途**：存储用户基本信息和状态

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| _id | String | 是 | 自动生成的唯一ID | "5f8a..." |
| openid | String | 是 | 微信用户唯一标识 | "oXXXX..." |
| unionid | String | 否 | 开放平台唯一标识 | "uXXXX..." |
| nickname | String | 是 | 用户昵称 | "张三" |
| avatar | String | 否 | 头像URL | "https://..." |
| isVIP | Boolean | 是 | 是否VIP会员 | false |
| vipExpireAt | Date | 否 | VIP过期时间 | "2027-03-02" |
| streakDays | Number | 是 | 连续打卡天数 | 7 |
| totalPoints | Number | 是 | 总积分 | 120 |
| treeStage | Number | 是 | 树木成长阶段(0-5) | 1 |
| createdAt | Date | 是 | 创建时间 | "2026-03-02T..." |
| updatedAt | Date | 是 | 更新时间 | "2026-03-02T..." |

**索引**：
```javascript
// 唯一索引
db.collection('users').createIndex({
  openid: 1
}, { unique: true })
```

**示例数据**：
```json
{
  "_id": "5f8a7b2c3d4e5f6g7h8i9j0k",
  "openid": "oXXXX1234567890",
  "unionid": "uXXXX1234567890",
  "nickname": "快乐的小明",
  "avatar": "https://wx.qlogo.cn/...",
  "isVIP": false,
  "vipExpireAt": null,
  "streakDays": 7,
  "totalPoints": 120,
  "treeStage": 1,
  "createdAt": "2026-02-24T08:00:00.000Z",
  "updatedAt": "2026-03-02T08:00:00.000Z"
}
```

### 2. records（好事记录表）

**用途**：存储用户的每日好事记录

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| _id | String | 是 | 自动生成的唯一ID | "5f8a..." |
| openid | String | 是 | 用户标识 | "oXXXX..." |
| userId | String | 是 | 用户ID（关联users表） | "5f8a..." |
| content | String | 是 | 好事内容 | "今天天气很好" |
| category | String | 是 | 分类 | "sensory" |
| imageUrl | String | 否 | 图片URL | "cloud://..." |
| voiceUrl | String | 否 | 语音URL | "cloud://..." |
| aiFeedback | String | 是 | AI反馈 | "真好啊..." |
| points | Number | 是 | 获得积分 | 5 |
| date | String | 是 | 日期(YYYY-MM-DD) | "2026-03-02" |
| isPublic | Boolean | 是 | 是否公开到社区 | false |
| resonanceCount | Number | 否 | 共鸣数 | 0 |
| createdAt | Date | 是 | 创建时间 | "2026-03-02T..." |

**索引**：
```javascript
// 组合索引：用户+日期（用于查询今日记录）
db.collection('records').createIndex({
  openid: 1,
  date: -1
})

// 组合索引：公开+时间（用于社区查询）
db.collection('records').createIndex({
  isPublic: 1,
  createdAt: -1
})

// 单字段索引：用户ID
db.collection('records').createIndex({
  userId: 1
})
```

**分类枚举**：
- `interpersonal` - 人际
- `achievement` - 成就
- `sensory` - 感官
- `flow` - 心流
- `gratitude` - 感恩
- `other` - 其他

**示例数据**：
```json
{
  "_id": "5f8a7b2c3d4e5f6g7h8i9j0k",
  "openid": "oXXXX1234567890",
  "userId": "5f8a7b2c3d4e5f6g7h8i9j0k",
  "content": "今天早上看到窗外的阳光特别温暖，让我想起了小时候的夏天",
  "category": "sensory",
  "imageUrl": "",
  "voiceUrl": "",
  "aiFeedback": "真好啊，这样的小确幸值得被记住 ✨",
  "points": 7,
  "date": "2026-03-02",
  "isPublic": false,
  "resonanceCount": 0,
  "createdAt": "2026-03-02T08:30:00.000Z"
}
```

### 3. achievements（成就表）

**用途**：记录用户解锁的成就徽章

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| _id | String | 是 | 自动生成的唯一ID | "5f8a..." |
| openid | String | 是 | 用户标识 | "oXXXX..." |
| userId | String | 是 | 用户ID | "5f8a..." |
| badgeId | String | 是 | 徽章ID | "fire_habit" |
| badgeName | String | 是 | 徽章名称 | "火苗习惯" |
| unlockedAt | Date | 是 | 解锁时间 | "2026-03-02T..." |

**索引**：
```javascript
// 组合索引：用户+徽章（防止重复解锁）
db.collection('achievements').createIndex({
  openid: 1,
  badgeId: 1
}, { unique: true })
```

**示例数据**：
```json
{
  "_id": "5f8a7b2c3d4e5f6g7h8i9j0k",
  "openid": "oXXXX1234567890",
  "userId": "5f8a7b2c3d4e5f6g7h8i9j0k",
  "badgeId": "fire_habit",
  "badgeName": "火苗习惯",
  "unlockedAt": "2026-03-02T08:00:00.000Z"
}
```

### 4. resonance（共鸣表）

**用途**：记录用户对社区内容的共鸣（点赞）

**字段说明**：

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| _id | String | 是 | 自动生成的唯一ID | "5f8a..." |
| openid | String | 是 | 用户标识 | "oXXXX..." |
| recordId | String | 是 | 记录ID | "5f8a..." |
| createdAt | Date | 是 | 创建时间 | "2026-03-02T..." |

**索引**：
```javascript
// 组合索引：用户+记录（防止重复共鸣）
db.collection('resonance').createIndex({
  openid: 1,
  recordId: 1
}, { unique: true })

// 单字段索引：记录ID（用于统计共鸣数）
db.collection('resonance').createIndex({
  recordId: 1
})
```

## 🔐 权限设置

### users 集合
```json
{
  "read": "auth && doc.openid == auth.openid",
  "write": "auth && doc.openid == auth.openid"
}
```

### records 集合
```json
{
  "read": "auth && (doc.openid == auth.openid || doc.isPublic == true)",
  "write": "auth && doc.openid == auth.openid"
}
```

### achievements 集合
```json
{
  "read": "auth && doc.openid == auth.openid",
  "write": false
}
```
*注：成就只能通过云函数写入*

### resonance 集合
```json
{
  "read": "auth",
  "write": "auth && doc.openid == auth.openid"
}
```

## 📈 数据统计查询

### 1. 用户总数
```javascript
db.collection('users').count()
```

### 2. 今日活跃用户（DAU）
```javascript
const today = new Date().toISOString().split('T')[0]
db.collection('records').where({
  date: today
}).field({
  openid: true
}).get()
// 然后去重统计
```

### 3. VIP 用户数
```javascript
db.collection('users').where({
  isVIP: true
}).count()
```

### 4. 总记录数
```javascript
db.collection('records').count()
```

### 5. 用户连续打卡排行
```javascript
db.collection('users')
  .orderBy('streakDays', 'desc')
  .limit(100)
  .get()
```

### 6. 最受欢迎的好事（社区）
```javascript
db.collection('records')
  .where({
    isPublic: true
  })
  .orderBy('resonanceCount', 'desc')
  .limit(20)
  .get()
```

## 🔄 数据生命周期

### 数据保留策略

1. **用户数据**：永久保留
2. **好事记录**：永久保留
3. **成就记录**：永久保留
4. **共鸣记录**：永久保留

### 数据备份

建议在云开发控制台设置：
- **自动备份**：每天凌晨 3 点
- **保留天数**：30 天
- **备份到**：云存储

## 📊 容量估算

### 单条记录大小

- **users**：约 500 字节
- **records**：约 300 字节（不含图片/语音）
- **achievements**：约 200 字节
- **resonance**：约 100 字节

### 10,000 用户估算

假设每个用户：
- 平均记录 100 天
- 每天 3 条记录
- 解锁 5 个成就
- 产生 50 次共鸣

**总容量**：
- users: 10,000 × 500B = 5MB
- records: 10,000 × 100 × 3 × 300B = 900MB
- achievements: 10,000 × 5 × 200B = 10MB
- resonance: 10,000 × 50 × 100B = 50MB

**合计**：约 965MB（不含图片/语音）

## 🚀 性能优化

### 1. 索引优化
- 为高频查询字段创建索引
- 使用组合索引减少查询时间
- 定期检查索引使用情况

### 2. 查询优化
- 使用 `field()` 只返回需要的字段
- 使用 `limit()` 限制返回数量
- 避免全表扫描

### 3. 数据分片
- 当单个集合超过 10GB 时考虑分片
- 按日期或用户 ID 分片

## 🔧 维护脚本

### 清理测试数据
```javascript
// 删除测试用户
db.collection('users').where({
  nickname: db.RegExp({
    regexp: '^测试',
    options: 'i'
  })
}).remove()
```

### 统计数据质量
```javascript
// 查找空内容的记录
db.collection('records').where({
  content: ''
}).count()
```

### 修复数据
```javascript
// 修复缺失的 resonanceCount
db.collection('records').where({
  resonanceCount: _.exists(false)
}).update({
  data: {
    resonanceCount: 0
  }
})
```
