# 🚀 快速部署指南

## 方案一：一键部署（推荐）

### 1. 部署后端到 Railway

1. **访问 Railway**
   - 打开 https://railway.app/
   - 使用 GitHub 账号登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择 `three-good-things` 仓库

3. **添加 MongoDB**
   - 点击 "+ New"
   - 选择 "Database" → "Add MongoDB"
   - 等待数据库创建完成

4. **配置后端服务**
   - 点击后端服务
   - 进入 "Settings" → "Root Directory"
   - 设置为 `/backend`

5. **配置环境变量**
   - 进入 "Variables" 标签
   - 添加以下变量：
     ```
     NODE_ENV=production
     PORT=3000
     JWT_SECRET=your-random-secret-key-here
     MONGODB_URI=${{MongoDB.MONGO_URL}}
     FRONTEND_URL=https://your-app.vercel.app
     ```
   - 可选（如需 AI 功能）：
     ```
     OPENAI_API_KEY=sk-your-openai-key
     ```

6. **部署**
   - 点击 "Deploy"
   - 等待部署完成
   - 记录你的后端 URL：`https://your-app.railway.app`

---

### 2. 部署前端到 Vercel

1. **访问 Vercel**
   - 打开 https://vercel.com/
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "Add New" → "Project"
   - 选择 `three-good-things` 仓库
   - 点击 "Import"

3. **配置构建设置**
   - Framework Preset: `Other`
   - Build Command: `npm run build:h5`
   - Output Directory: `dist/build/h5`
   - Install Command: `npm install`

4. **配置环境变量**
   - 点击 "Environment Variables"
   - 添加：
     ```
     VITE_API_URL=https://your-app.railway.app/api
     ```
   - 将 `your-app.railway.app` 替换为你的 Railway 后端地址

5. **部署**
   - 点击 "Deploy"
   - 等待部署完成
   - 你会得到一个 URL：`https://your-app.vercel.app`

6. **更新后端配置**
   - 回到 Railway
   - 更新 `FRONTEND_URL` 为你的 Vercel 地址
   - 重新部署后端

## 方案二：使用 Netlify（前端替代方案）

### 部署前端到 Netlify

1. **访问 Netlify**
   - 打开 https://www.netlify.com/
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "Add new site" → "Import an existing project"
   - 选择 GitHub
   - 选择 `three-good-things` 仓库

3. **配置构建**
   - Build command: `npm run build:h5`
   - Publish directory: `dist/build/h5`
   - 环境变量：
     ```
     VITE_API_URL=https://your-app.railway.app/api
     ```

4. **部署**
   - 点击 "Deploy site"
   - 等待完成

## 方案三：国内部署

### 使用阿里云/腾讯云

#### 后端部署

1. **购买云服务器**
   - 最低配置：1核2G
   - 系统：Ubuntu 20.04

2. **安装环境**
   ```bash
   # 安装 Node.js
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs

   # 安装 MongoDB
   sudo apt-get install -y mongodb

   # 安装 PM2
   sudo npm install -g pm2
   ```

3. **部署代码**
   ```bash
   # 克隆代码
   git clone https://github.com/your-username/three-good-things.git
   cd three-good-things/backend

   # 安装依赖
   npm install

   # 配置环境变量
   cp .env.example .env
   nano .env  # 编辑配置

   # 构建
   npm run build

   # 启动
   pm2 start npm --name "three-good-things" -- start
   pm2 save
   pm2 startup
   ```

4. **配置 Nginx**
   ```nginx
   server {
       listen 80;
       server_name api.yourdomain.com;

       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

#### 前端部署

1. **构建前端**
   ```bash
   npm run build:h5
   ```

2. **上传到 OSS**
   - 使用阿里云 OSS 或腾讯云 COS
   - 上传 `dist/build/h5` 目录
   - 配置静态网站托管
   - 绑定自定义域名

## 常见问题

### Q: 部署后前端无法连接后端？
**A:** 检查 `VITE_API_URL` 是否正确，确保包含 `/api` 后缀

### Q: Railway 部署失败？
**A:** 检查 Root Directory 是否设置为 `/backend`

### Q: MongoDB 连接失败？
**A:** 确认 MongoDB 服务已启动，连接字符串正确

### Q: 如何查看后端日志？
**A:** Railway: 点击服务 → "Deployments" → 查看日志
