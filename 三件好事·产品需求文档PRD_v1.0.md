# 一、产品概述

## 1.1 产品愿景

> ### 💬 「 让更多人养成感知美好的能力，在生活的细微处发现幸福 」
> *— Three Good Things 产品使命宣言*

《三件好事》是一款基于积极心理学「三件好事」干预理论（Martin Seligman, 2005）与游戏化行为设计（Octalysis八角框架）的情绪健康记录产品。用户通过每日记录三件好事，养育专属虚拟生命，在轻量社区中感受共鸣，逐步建立感恩习惯，提升主观幸福感。

| 维度 | 详情 |
|------|------|
| 🎯 目标用户 | 2.3亿 中国成年焦虑人群 |
| 📖 核心理论 | 积极心理学三件好事（PPT） |
| 🎮 游戏化框架 | Octalysis八角驱动力体系 |
| 📱 产品形态 | iOS + Android + 小程序 |

---

## 1.2 产品定位

### 核心差异化

1. **最轻的记录门槛**（目标：30秒完成三件好事）
2. **最有温度的AI陪伴**（像朋友，不像工具）
3. **最克制的社交设计**（匿名社区，无焦虑比较）
4. **与感情深度绑定的虚拟生命体验**

### 竞品对比

| 维度 | 三件好事 | Daylio | 一言一记 | Forest |
|------|----------|--------|----------|--------|
| 理论背书 | ✅ 积极心理学 | ❌ 无 | ⚠️ 弱 | ⚠️ 专注 |
| AI分析 | ✅ 深度 | ❌ 无 | ❌ 无 | ❌ 无 |
| 游戏化 | ✅ 八角体系 | ⚠️ 简单 | ❌ 无 | ✅ 种树 |
| 社区功能 | ✅ 匿名温暖 | ❌ 无 | ❌ 无 | ⚠️ 弱 |

> 🎯 **主战场**：情绪健康 × 自我成长赛道

- **TAM**：中国成年人情绪困扰群体约 **2.3亿**
- **SAM**：对心理健康有主动认知的用户约 **6000万**
- **SOM**：18-35岁，一二线城市，接受付费订阅约 **800万**

## 2.3 北极星指标

> ⭐ **核心 NSM**：用户连续记录天数（Streak Days）中位数

**选择原因**：连续记录天数与用户幸福感提升正相关（r=0.67，Seligman研究数据），且与留存率强相关。

| 指标 | 目标值 |
|------|--------|
| Day-7 留存率 | ≥ 45% |
| Day-30 留存率 | ≥ 25% |
| 连续记录7天用户占比 | ≥ 30% |

# 三、核心功能需求

## 3.1 快速录入系统

> **设计原则**：用户从打开App到完成三件好事记录，全程不超过90秒；单件好事录入不超过30秒。

### 3.1.1 主记录界面设计规范

- 进入App后，主页面首先展示今日记录状态（已记录N件/3件），视觉上清晰传达进度
- 三个输入槽位顺序排列，当前激活槽位有轻微呼吸动效（scale: 1.0→1.02，周期2s）
- 输入框默认展示AI生成的动态引导语，每日轮换，降低空白恐惧感
- 键盘弹出时页面自动上移，保证输入框可见；底部工具栏（语音/相机/表情）随键盘出现

### 3.1.2 用户故事

> **👤 小雅 · 上班族 · 通勤场景**
>
> 作为一个想培养感恩习惯的用户，我希望能在地铁上30秒内快速记录一件好事，不需要思考太多，记录完就有小小的成就感。

**验收标准：**

- ✓ 打开App到看见输入框，加载时间 < 1.5秒
- ✓ 输入框自动获得焦点（iOS弹出键盘，Android可选自动/手动）
- ✓ 输入内容无字数下限，1个字也可以提交
- ✓ 提交后有即时正向反馈（AI回应 + 树木微动效），耗时 < 800ms
- ✓ 支持离线记录，网络恢复后自动同步

### 3.1.3 输入方式

| 输入方式 | 入口 | 交互流程 | 优先级 | 备注 |
|----------|------|----------|--------|------|
| 文字输入 | 主输入框 | 点击输入框 → 键盘弹出 → 输入 → 点击「记录」 | **P0** | 基础功能，必须上线 |
| 语音转录 | 键盘上方工具栏麦克风 | 长按麦克风 → 说话 → 松手 → AI整理 → 确认 | **P0** | 需要系统语音权限 |
| 快捷词标签 | 输入框下方快捷标签栏 | 点击预设好事标签 → 一键填入 → 可编辑 | **P1** | 标签基于用户历史推荐 |
| 复制社区好事 | 社区页面「复制」按钮 | 点击复制 → AI轻度个性化 → 跳转录入页确认 | **P1** | 需注意版权归属提示 |
| 照片关联 | 键盘上方相机图标 | 选择/拍摄照片 → 附加到当前好事记录（可选描述） | **P2** | MVP后续迭代 |

### 3.1.4 AI即时反馈

每次用户提交一件好事后，AI在**800ms内**返回即时反馈，由三部分组成：

| 组成 | 说明 |
|------|------|
| ① 情感呼应句 | 1-2句温暖友好的回应，避免套话，像朋友的随手一句话 |
| ② 分类标签 | 人际 / 成就 / 感官 / 心流 / 感恩 / 其他，自动识别 |
| ③ 树木积分动效 | +N积分数字浮出，树木叶片轻颤动效，成就感即时反馈 |

**异常处理规则：**

> ⚠️ **网络超时**：缓存记录本地，显示「已保存，稍后上传」，不显示AI反馈，树木积分待网络恢复后补算
>
> ⚠️ **AI服务降级**：使用预设温暖文案库（本地缓存50条）随机呈现，保证基础体验不中断
>
> ⚠️ **重复内容检测**：同一用户7天内若出现高度相似内容，轻提示「你最近经常感受到这种美好~」，不阻止提交

## 3.2 AI智能分析系统

### 3.2.1 实时情感识别（静默后台）

每次提交三件好事后，后台**静默运行**情感识别，结果不直接呈现给用户，用于驱动推送策略和树木状态更新。

| 识别维度 | 判断逻辑 | 输出结果 | 触发动作 |
|----------|----------|----------|----------|
| 情感基调 | 分析措辞积极度、内容密度、细节丰富度 | 积极/平稳/略显低落 | 决定次日晨间推送语气 |
| 记录质量 | 字数、情感投入度、具体程度 | 1-10分 | 计算树木成长积分 |
| 关怀触发 | 连续N天无记录、内容消极、字数极短 | true/false | 触发关怀推送模板 |
| 雷达主题 | 基于当日内容推断明日可关注方向 | 主题标签 | 次日晨间推送附带建议 |

### 3.2.2 每周分析报告

> 📅 **推送时机**：每周一早上8:00（用户最近一次记录时间的所在周结束时推送）
> 若用户本周记录少于3天，推送改为「轻量版」（只有亮点句 + 邀请继续），不推送完整分析

| 报告模块 | 内容说明 | 数据来源 | 展示形式 |
|----------|----------|----------|----------|
| 本周亮点句 | 一句话捕捉本周最闪光的时刻，AI生成 | 本周所有记录 | 大号字体，居中展示 |
| 好事分类图 | 六大分类的占比环形图 | 分类统计 | 可交互环形图 |
| 模式洞察 | 哪些领域收获最多，有何规律 | 历史对比数据 | 2-3句文字 + 关键词高亮 |
| 成长信号 | 对比上周变化，有什么新的出现 | 周环比数据 | ↑↓ 趋势指标卡 |
| 下周轻建议 | 一条邀请式建议，非任务 | AI生成 | 小卡片，可收藏 |
| 树木周报 | 本周树木成长状态（与记录频率挂钩） | 积分系统 | 树木插画 + 一句诗意描述 |

### 3.2.3 月度深度报告

**数据维度：**
- 记录总数 & 坚持天数
- 六大分类分布趋势图
- 高频关键词词云
- 最佳记录时段热力图
- 月积分 & 树木成长历程

**叙事内容：**
- 你的幸福密码（核心发现，用具体例子说话）
- 你最幸福的时刻类型
- 你的成长轨迹（月内变化趋势）
- 给下个月的你一封信（200字以内）
- 月度最美好的一天（单日记录最丰富）

## 3.3 虚拟生命系统

> **核心设计理念**：虚拟生命的成长状态与用户的好事记录行为深度绑定，形成情感寄托，驱动持续回访。参考支付宝蚂蚁森林的成功经验，但更侧重情感温度而非功能完整度。

### 3.3.1 生命形态体系

| 阶段 | 名称 | 解锁积分 | 参考天数 | 成长叙事触发 |
|------|------|----------|----------|--------------|
| 🌰 | 种子 | 0 pts | 第1天 | 开始埋下一颗期待 |
| 🌱 | 嫩芽 | 50 pts | 约7天 | 第一片嫩叶冒出，有了生命的痕迹 |
| 🌿 | 幼苗 | 200 pts | 约21天 | 根系开始延伸，习惯正在扎根 |
| 🌳 | 小树 | 500 pts | 约45天 | 枝叶渐茂，可以给自己一点荫凉了 |
| 🌲 | 大树 | 1200 pts | 约90天 | 四季轮换，见证了你许多个清晨 |
| 🌴 | 参天古树 | 3000 pts | 约180天 | 一棵属于你的，关于美好的纪念碑 |

### 3.3.2 积分计算规则

| 积分来源 | 基础分值 | 加成条件 | 加成 |
|----------|----------|----------|------|
| 记录一件好事 | +5分 | 内容超过20字 | +2分 |
| 完成当日三件好事 | +15分（额外） | 同日均超过30字 | +5分额外奖励 |
| 连续记录加成 | +0分 | 连续7/14/21/30天 | ×1.2 / ×1.5 / ×2.0 / ×2.5 |
| 分类多样性 | +0分 | 一周内覆盖4种以上分类 | +20分（周结算） |
| 社区内容被复制 | +3分 | 同一内容被复制5次以上 | +额外10分 |
| 里程碑解锁 | 固定分值 | 见成就徽章体系 | — |

### 3.3.3 特殊事件系统

除常规成长外，系统随机触发特殊事件，增加未知感与惊喜感（Octalysis第7驱动力）：

| 事件 | 名称 | 触发条件 |
|------|------|----------|
| 🌧️ | 雨后彩虹 | 连续记录后遇到第一个周末，树木出现彩虹特效 |
| 🍂 | 季节变换 | 真实节气触发，树木出现对应季节形态变化 |
| 🐦 | 小鸟造访 | 连续21天触发，小鸟在树上筑巢动画 |
| ✨ | 夜间萤火 | 用户在23:00后记录，触发萤火虫环绕特效 |

## 3.4 温暖社区系统

### 3.4.1 社区设计原则

**✅ 我们鼓励：**
- 匿名分享，默认保护隐私
- 只能点击「✨共鸣」，不能评论（降低社交焦虑）
- 一键复制他人好事到自己记录
- 显示「今日X人感受到类似的美好」
- 每日社区精选推送3条（算法筛选最有温度）

**❌ 我们禁止：**
- 显示用户名/头像（除非用户主动公开）
- 评论、回复、@他人
- 点赞数排名显示（避免比较焦虑）
- 含有商业推广、负面情绪的内容进入精选
- 含有可识别个人信息的内容公开展示

### 3.4.2 内容展示逻辑

| 展示位置 | 内容来源 | 筛选算法 | 刷新频率 |
|----------|----------|----------|----------|
| 首页今日精选 | 当日公开/匿名好事 | warmth_score ≥4 + 与用户历史类别交集 | 每4小时 |
| 探索流（无限滚动） | 近7天公开内容 | 时间衰减 × 共鸣数 × 多样性 | 实时 |
| 相似好事推荐 | 记录完成后触发 | 语义相似度 + 用户画像匹配 | 按需 |
| 社区日历 | 历史公开内容按日归档 | 用户自选日期查看当日社区 | 静态 |

### 3.4.3 用户故事

> **👤 小雅 · 浏览社区 · 睡前场景**
>
> 作为一个注重隐私的用户，我希望能看到其他人真实温暖的好事分享，找到共鸣感，同时不需要表达自己的意见，也不担心被人评判。

**验收标准：**
- ✓ 社区流中不显示任何可识别用户身份的信息
- ✓ 「共鸣」按钮点击有轻微触感反馈（haptic），无需确认
- ✓ 「复制」后跳转录入页，内容预填但可编辑，需再次主动点击提交
- ✓ 用户可以设置「不看来自某类别的好事」进行内容过滤
- ✓ 每次打开探索流，前3条内容不重复（新鲜感保证）

# 四、游戏化设计详述（Octalysis八角体系）

## 4.1 八角驱动力全景映射

| # | 驱动力 | 产品实现 | 关键体验节点 | 衡量指标 |
|---|--------|----------|--------------|----------|
| 1 | 史诗意义感 | 积极心理学科学背书 + 树木成长叙事诗意化 | 新手引导 · 里程碑对话 · 树木生长描述 | 新用户首日完成率 |
| 2 | 成就与自豪 | 成就徽章系统 + 连击记录 + 月度分析报告 | 徽章解锁 · 连击达成 · 报告接收 | 7天留存率 |

# 五、智能推送系统

## 5.1 推送策略总览

> 📣 **核心原则**
> - **宁少勿多**：默认推送频率不超过每日1条，用户可自定义
> - **宁晚勿扰**：推送时间段限制在7:30-22:00之间
> - **个性化优先**：基于用户行为数据动态调整推送内容和时机
> - **情感安全**：识别到用户情绪低落时，推送语气更温柔，降低行动门槛

| 推送类型 | 触发条件 | 发送时间 | 文案风格 | 优先级 |
|----------|----------|----------|----------|--------|
| 晨间唤醒 | 每日（可关闭） | 用户自设，默认8:00 | 今日天气/节气/冷知识开场 + 轻柔邀请 | 可选 |
| 晚间提醒 | 当日未记录 + 20:00 | 20:00 | 围绕树木想念，30字内，不焦虑 | **核心** |
| 连击保护 | 连击≥7天 + 当日18:00未记录 | 18:00 | 温暖不焦虑，强调连击价值 | **核心** |
| 里程碑庆祝 | 7/21/66/100天等解锁 | 解锁后即时 | 个性化叙事，回顾高光时刻 | **核心** |
| 关怀推送 | 连续3天未记录 | 14:00 | 不追问，表达树木想念，降低重启门槛 | **核心** |
| 周报推送 | 每周一，上周记录≥3天 | 8:00 | 附带分析摘要预览，吸引打开 | **核心** |
| 社区热点 | 高共鸣内容出现 | 14:00-16:00 | 「今日X人共同感受到了……」 | 可选 |

## 6.2 隐私保护规范

### 数据分级

| 级别 | 数据类型 | 存储/处理规范 |
|------|----------|---------------|
| L1 最敏感 | 好事原始文本内容 | 服务端加密存储，AI处理后不留原文日志 |
| L2 敏感 | 用户行为数据、情感评分 | 脱敏处理，用于内部分析，不传第三方 |
| L3 一般 | 积分、成就、树木状态 | 常规存储，用于产品功能展示 |
| L4 公开 | 用户主动分享的社区内容 | 展示前经过AI匿名化处理 |

### 🔐 隐私五大承诺

1. **不出售**用户数据给任何第三方
2. AI分析**只用于个人报告**，不用于用户画像商业化
3. 用户可随时**导出全部数据**（JSON格式）
4. 用户可**一键删除**账号及全部记录
5. 社区匿名内容**不可被追溯**到具体用户
