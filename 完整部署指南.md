## 🚀 快速开始

### 1. 后端部署

#### 本地开发

```bash
# 进入后端目录
cd backend

# 安装依赖
npm install

# 复制环境变量
cp .env.example .env

# 编辑 .env 文件，填入配置
# - MONGODB_URI: MongoDB 连接字符串
# - JWT_SECRET: JWT 密钥
# - GOOGLE_CLIENT_ID: Google OAuth ID
# - GOOGLE_CLIENT_SECRET: Google OAuth Secret
# - OPENAI_API_KEY: OpenAI API Key

# 启动开发服务器
npm run dev
```

服务器将运行在 `http://localhost:3000`

#### 生产部署（Railway）

1. 在 [Railway](https://railway.app/) 创建新项目
2. 添加 MongoDB 服务
3. 连接 GitHub 仓库
4. 设置环境变量：
   - `MONGODB_URI`
   - `JWT_SECRET`
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - `OPENAI_API_KEY`
   - `FRONTEND_URL`
5. 部署

### 2. 前端部署

#### 本地开发

```bash
# 在项目根目录

# 编辑 .env 文件
# VITE_API_URL=http://localhost:3000/api

# 启动 H5 开发服务器
npm run dev:h5
```

访问 `http://localhost:5173`

#### 生产部署（Vercel）

```bash
# 构建生产版本
npm run build:h5

# 部署到 Vercel
# 1. 在 Vercel 创建新项目
# 2. 连接 GitHub 仓库
# 3. 设置环境变量：
#    - VITE_API_URL=https://your-api.railway.app/api
#    - VITE_GOOGLE_CLIENT_ID=your-google-client-id
# 4. 构建命令：npm run build:h5
# 5. 输出目录：dist/build/h5
```

## 🔐 Google OAuth 配置

### 1. 创建 Google OAuth 应用

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)
2. 创建新项目或选择现有项目
3. 启用 "Google+ API"
4. 创建 OAuth 2.0 凭据：
   - 应用类型：Web 应用
   - 授权重定向 URI：
     - 开发：`http://localhost:3000/api/auth/google/callback`
     - 生产：`https://your-api.railway.app/api/auth/google/callback`
5. 获取 Client ID 和 Client Secret

### 2. 配置环境变量

后端 `.env`:
```env
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
GOOGLE_CALLBACK_URL=http://localhost:3000/api/auth/google/callback
```

前端 `.env`:
```env
VITE_GOOGLE_CLIENT_ID=your-client-id
```

## 💾 MongoDB 配置

### 本地开发

```bash
# 安装 MongoDB
# Windows: https://www.mongodb.com/try/download/community
# Mac: brew install mongodb-community
# Linux: apt-get install mongodb

# 启动 MongoDB
mongod

# 连接字符串
MONGODB_URI=mongodb://localhost:27017/three-good-things
```

### 生产环境（MongoDB Atlas）

1. 访问 [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. 创建免费集群
3. 创建数据库用户
4. 获取连接字符串
5. 配置 IP 白名单（允许所有：0.0.0.0/0）

```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/three-good-things
```

## 🔧 常见问题

### Q1: 后端启动失败

**解决方案**：
- 检查 MongoDB 是否运行
- 检查 `.env` 配置是否正确
- 查看错误日志

### Q2: Google 登录失败

**解决方案**：
- 检查 Google OAuth 配置
- 确认回调 URL 正确
- 检查 Client ID 和 Secret

### Q3: AI 对话无响应

**解决方案**：
- 检查 OpenAI API Key 是否有效
- 确认账户有余额
- 查看后端日志

### Q4: 前端无法连接后端

**解决方案**：
- 检查 `VITE_API_URL` 配置
- 确认后端服务运行中
- 检查 CORS 配置

## 📊 API 端点

### 认证
- `POST /api/auth/register` - 注册
- `POST /api/auth/login` - 登录
- `GET /api/auth/google` - Google 登录
- `POST /api/auth/guest` - 游客登录
- `POST /api/auth/upgrade` - 升级账号

### 用户
- `GET /api/user/profile` - 获取用户信息
- `PUT /api/user/profile` - 更新用户信息

### 记录
- `POST /api/records` - 创建记录
- `GET /api/records` - 获取记录列表
- `GET /api/records/today` - 获取今日记录
- `DELETE /api/records/:id` - 删除记录

### AI
- `POST /api/ai/chat` - AI 对话

## 🔒 安全建议

1. **生产环境必须**：
   - 使用 HTTPS
   - 更改默认 JWT_SECRET
   - 限制 CORS 来源
   - 启用速率限制

2. **数据库安全**：
   - 使用强密码
   - 限制 IP 访问
   - 定期备份

3. **API 安全**：
   - 输入验证
   - SQL 注入防护
   - XSS 防护
