## 📊 数据库设计

### 集合列表

| 集合名 | 用途 | 状态 |
|--------|------|------|
| users | 用户信息 | ✅ 已设计 |
| records | 好事记录 | ✅ 已设计 |
| achievements | 成就徽章 | ✅ 已设计 |
| resonance | 共鸣记录 | ✅ 已设计 |

### 数据流程

```
用户登录
  ↓
调用 user 云函数
  ↓
创建/更新 users 表
  ↓
返回用户信息
  ↓
用户提交好事
  ↓
调用 record 云函数
  ↓
写入 records 表
  ↓
更新 users 表（积分、树木阶段）
  ↓
返回 AI 反馈和积分
```

## 🚀 部署步骤（简化版）

### 1. 开通云开发
在微信开发者工具中开通云开发，获取环境 ID

### 2. 配置环境 ID
修改 `src/api/cloud.ts`：
```typescript
wx.cloud.init({
  env: 'your-env-id', // 替换这里
  traceUser: true
})
```

### 3. 创建数据库集合
在云开发控制台创建：
- users
- records
- achievements（可选）
- resonance（可选）

### 4. 上传云函数
右键点击云函数目录 → 上传并部署：云端安装依赖
- cloudfunctions/user
- cloudfunctions/record

### 5. 切换到云开发模式
```bash
# 备份原文件
mv src/store/user.ts src/store/user-http.ts

# 使用云开发版本
mv src/store/user-cloud.ts src/store/user.ts
```

### 6. 测试
编译运行小程序，测试登录和记录功能

## 💰 成本估算

### 免费额度（每月）
- 数据库读：50,000 次
- 数据库写：30,000 次
- 云函数调用：100,000 次
- 存储：2GB

### 10,000 用户估算
假设每个用户每天：
- 登录 1 次
- 记录 3 件好事
- 查询历史 2 次

**每月消耗**：
- 云函数调用：10,000 × 30 × 6 = 1,800,000 次（超出免费额度）
- 数据库写：10,000 × 30 × 3 = 900,000 次（超出免费额度）
- 数据库读：10,000 × 30 × 3 = 900,000 次（超出免费额度）

**预估费用**：约 ¥100-200/月

### 优化建议
1. 使用本地缓存减少云函数调用
2. 批量操作减少数据库读写
3. 使用 CDN 缓存静态资源
